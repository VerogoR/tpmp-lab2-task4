# Отчёт: контрольные расчёты и тесты

## Цель

Проверка работоспособности программы «Абитуриент»: чтение данных из текстового файла, фильтрация абитуриентов с проходным баллом >= 225, сохранение результата в новый файл.

## Структура проекта (модель КИС)

- **Клиент** (`src/main.c`) — организация просмотра файла, вызов сервера для чтения/фильтрации/записи, вывод результатов.
- **Интерфейс** (`include/applicant.h`) — объявление типов и функций.
- **Сервер** (`src/applicant_server.c`) — реализация чтения из файла, фильтрации по проходному баллу, записи в файл.

## Формат входного файла

Один абитуриент — блок из 16 строк:

1. Фамилия  
2. Имя  
3. Отчество  
4. Пол (М/Ж)  
5. Национальность  
6. Дата рождения (год месяц день)  
7. Почтовый индекс  
8. Страна  
9. Область  
10. Район  
11. Город  
12. Улица  
13. Дом  
14. Квартира  
15. Баллы ЦТ (через пробел)  
16. Проходной балл  

## Исходные данные (контрольный ввод)

Файл `tests/applicants.txt`:

| № | Фамилия   | Имя     | Проходной балл |
|---|-----------|---------|----------------|
| 1 | Иванов    | Иван    | 275            |
| 2 | Петрова   | Анна    | 240            |
| 3 | Сидоров   | Петр    | 203            |
| 4 | Козлова   | Мария   | 290            |

Ожидаемая выборка (проходной балл >= 225): **Иванов, Петрова, Козлова**. Сидоров (203) не попадает в результат.

## Выполнение теста

```bash
make
cp tests/applicants.txt applicants.txt
./applicants applicants.txt result.txt
```

## Ожидаемые результаты

1. **Просмотр файла** — программа выводит содержимое `applicants.txt`.
2. **Чтение** — прочитано 4 абитуриента.
3. **Фильтрация** — отобрано 3 абитуриента с проходным баллом >= 225.
4. **Файл результата** `result.txt` содержит только записи Иванова, Петровой и Козловой (в том же формате по 16 строк на каждого).
